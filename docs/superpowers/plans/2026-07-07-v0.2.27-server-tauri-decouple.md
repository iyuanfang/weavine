# v0.2.27 -- Decouple server from Tauri (fix prod glibc + buildability)

> For agentic workers: REQUIRED SUB-SKILL: superpowers:subagent-driven-development
> Steps use checkbox (- [ ]) syntax.

**Goal:** Make the weavine-server binary buildable on a host with old glibc (Aliyun Linux 8, glibc 2.32) by removing the unintended dependency on tauri -> muda -> gtk -> glib-sys. After this lands, dev/glibc-2.39 binaries and prod/glibc-2.32 binaries both build cleanly, and the v0.2.26 archive fix finally reaches the prod web deployment.

**Root cause (one paragraph):** server/Cargo.toml depends on weavine (path ../src-tauri). The weavine crate is a hybrid: it has both pure-Rust code (handlers, models, db, business) AND the Tauri desktop glue (run(), commands, startup_error). Because [features] default = [tauri] in src-tauri/Cargo.toml, building the workspace pulls in tauri for every consumer -- including the server binary -- even with default-features = false on the consumer side (workspace-unified features override). Tauri's dep chain tauri -> muda -> gtk-sys -> atk-sys -> glib-sys requires glib >= 2.70 at build time. Dev box has glib 2.79, prod has glib 2.68 (Aliyun Linux 3) and glibc 2.32 (Aliyun Linux 8). Result: dev-built server binaries need GLIBC 2.34+; prod cannot build because the glib pkg-config is too old.

**Architecture decision (already made):** Extract a weavine-core sub-crate from src-tauri/. Keep weavine (the Tauri desktop crate) as a thin Tauri shell depending on weavine-core. Make server/ depend on weavine-core directly. The pure-Rust handlers/models/db are already in modules with no tauri:: use; they just need to move to the new crate. The [[bin]] required-features = [tauri] guard is the belt-and-suspenders for future regressions.

**Spec / design doc:** This plan is self-contained. No separate spec needed (the design IS the refactor; behavior is preserved 1:1).

**Existing patterns to follow:**
- Workspace layout: 1 Cargo workspace at repo root, members = [src-tauri, server]. We will add a third member weavine-core/ and remove src-tauri/'s claim on the pure-Rust modules.
- sqlx 0.8 with runtime-tokio-rustls, postgres, chrono, uuid, migrate, macros -- server already has this exact setup; weavine-core just needs the same.
- Handler signature: async fn handler(headers, State(pool), ...) -> Result<Json<X>, (StatusCode, String)>. Already 100% tauri-free.
- AGPL-3.0 license header in all new files.

**Out of scope:**
- Any v0.2.26 archive fix is already merged on origin/main; we do not re-fix it. We are only making the fix deployable.
- Tauri <-> Web SPA code path (no change).
- Cloud sync engine (sync/ module) -- keep as-is, moves with the rest.
- iOS build (out of scope for prod server; only desktop Android is shipped).
